Cartesian Closed Categories

Cartesian Closed Categories (CCCs) are categories with binary products, exponentials, and a terminal object. Although Lean seems to define something like this, I don't really get it. So I'm defining CCCs myself from scratch to learn how it's done.

Binary Products

A binary product of two objects X‚ÇÅ and X‚ÇÇ in a category is an object called X‚ÇÅ √ó X‚ÇÇ with projections and a universal property: For every object Y and morphisms f‚ÇÅ : Y ‚ü∂ X‚ÇÅ and f‚ÇÇ : Y ‚ü∂ X‚ÇÇ there is a unique morphism f : Y ‚ü∂ X‚ÇÅ √ó X‚ÇÇ such that f ‚â´ œÄ‚ÇÅ = f‚ÇÅ and f ‚â´ œÄ‚ÇÇ = f‚ÇÇ. This basically says that f is the shortest way to make a pair.

To encode this in Lean, we make a typeclass requiring the instantiator to say what this unique morphism is, and call it pair. The properties pair·µ¢ record the above universal property, and the unique_pair property records the requirement the morphism is unique.

Pairs

Pair only describes how to take the product of morphisms with the same domain. The following method, which builds on pair, allows products of arbitary morphisms, which will be useful in defining exponentials later.

def prod_map.{u} {C : Type u} [Category C] [HasProduct C]
             {X‚ÇÅ Y‚ÇÅ X‚ÇÇ Y‚ÇÇ : C} (f‚ÇÅ : Y‚ÇÅ ‚ü∂ X‚ÇÅ) (f‚ÇÇ : Y‚ÇÇ ‚ü∂ X‚ÇÇ)

  let g‚ÇÇ : P ‚ü∂ X‚ÇÇ := œÄ‚ÇÇ ‚â´ f‚ÇÇ

Exponentials

The Exponential Object Z^Y in many cases can be thought of all functions from Y into Z. We need an evaluation function

eval : Z^Y √ó Y ‚Üí Z

which takes a function in Z^Y and applies it to Y to get an element of Z.

We also need a univeral property, which states that for any X and morphism g : X √ó Y ‚Üí Z there is a unique morphism Œªg : X ‚Üí Z^Y so that (Œªg √ó (ùüô Y)) ‚â´ eval = g.

X √ó Y -- Œªg√ó(ùüô Y) --> Z^Y √ó Y --eval--> Z | ‚àß | | +-------------------g------------------+

This basically says that any morphism from X √ó Y to Z has the form (Œªg √ó (ùüô Y)) ‚â´ eval. There isn't a simpler way to say it.

Typically, one defines a curry function so that

X √ó Y ‚ü∂ Z ---curry---> X ‚ü∂ Z^Y

so that Œªg = curry g √ó ùüô Y.
